## API Overview

Steve Wiseman Documentation provides a comprehensive REST API for programmatic access to all platform features. Whether you're building custom integrations, automating workflows, or syncing with other tools, our API offers full functionality.

<Callout kind="info">All API endpoints require authentication using API keys or OAuth tokens.</Callout>

## Authentication

<Tabs>
  <Tab title="API Keys" icon="key">
    Generate API keys from your account settings for server-to-server authentication.
    
    <Steps>
      <Step title="Generate Key" icon="key">
        Go to Settings > API Keys and click "Generate New Key".
        
        <CodeGroup tabs="JavaScript,cURL">
          ````javascript
const apiKey = 'sk-your-api-key-here';
const headers = {
  'Authorization': `Bearer ${apiKey}`,
  'Content-Type': 'application/json'
};
````
          ````bash
API_KEY="sk-your-api-key-here"
HEADERS="Authorization: Bearer $API_KEY\nContent-Type: application/json"
````
        </Step>
      <Step title="Use in Requests" icon="send">
        Include the API key in the Authorization header for all requests.
      </Step>
    </Steps>
  </Tab>
  <Tab title="OAuth" icon="lock">
    Use OAuth 2.0 for user-authorized access in client applications.
    
    <Expandable title="OAuth Flow" default-open="false">
      1. Redirect users to our authorization URL
      2. User grants permissions
      3. Receive authorization code
      4. Exchange for access token
    </Expandable>
  </Tab>
</Tabs>

## Core Endpoints

<ExpandableGroup>
  <Expandable title="Workspaces" default-open="true">
    Manage documentation workspaces and their settings.
    
    <CodeGroup tabs="JavaScript,Python">
      ````javascript
// Get all workspaces
const workspaces = await fetch('/api/workspaces', {
  headers: { 'Authorization': `Bearer ${apiKey}` }
}).then(r => r.json());

// Create a workspace
const newWorkspace = await fetch('/api/workspaces', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'New Project Docs',
    description: 'Documentation for new project'
  })
}).then(r => r.json());
````
      ````python
import requests

# Get all workspaces
response = requests.get('/api/workspaces', 
                       headers={'Authorization': f'Bearer {api_key}'})
workspaces = response.json()

# Create a workspace
workspace_data = {
    'name': 'New Project Docs',
    'description': 'Documentation for new project'
}
response = requests.post('/api/workspaces', 
                        json=workspace_data,
                        headers={'Authorization': f'Bearer {api_key}'})
new_workspace = response.json()
````
    </Expandable>

    <Callout kind="tip">Documents support Markdown formatting and can include images and attachments.</Callout>
  </Expandable>

    Manage user accounts, permissions, and team collaboration.
  </Expandable>
</ExpandableGroup>

## Rate Limits

<Callout kind="alert">API requests are subject to rate limits to ensure fair usage.</Callout>

| Plan | Requests per Hour | Requests per Minute |
|------|-------------------|---------------------|
| Free | 1,000 | 100 |
| Pro | 10,000 | 500 |
| Enterprise | 100,000 | 2,000 |

<Expandable title="Handling Rate Limits" default-open="false">
When you exceed rate limits, the API returns a 429 status code. Implement exponential backoff in your retry logic.
</Expandable>

## Error Handling

<ExpandableGroup>
  <Expandable title="Common Error Codes" default-open="false">
    - 400 Bad Request: Invalid parameters
    - 401 Unauthorized: Missing or invalid authentication
    - 403 Forbidden: Insufficient permissions
    - 404 Not Found: Resource doesn't exist
    - 429 Too Many Requests: Rate limit exceeded
    - 500 Internal Server Error: Platform issue
  </Expandable>
  <Expandable title="Error Response Format" default-open="false">
    All errors follow a consistent JSON structure with error codes and messages.
  </Expandable>
</ExpandableGroup>
